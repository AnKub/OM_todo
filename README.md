# OMTODO

Невеликий todo-застосунок у форматі monorepo: React + TypeScript на фронті, Express + TypeScript на бекенді.

З боку функціоналу тут реалізовано базовий, але чесний набір речей: завантаження списку задач з API, створення, редагування, видалення, фільтрація активних задач, збереження фільтра в localStorage, тести для ключових сценаріїв і стилізація з двома темами. Тобто без магії, без "ентерпрайзу на 3 задачі", але і без халтури.

Під час реалізації я тримався простих принципів: KISS, зрозуміла структура, мінімум зайвих абстракцій, читабельні назви, ізоляція відповідальностей і фокус на тому, щоб кожен шматок коду можна було нормально пояснити на співбесіді, а не лише показати, що він "десь там працює".

Бекенд я спеціально розділив на кілька шарів, щоб ролі не змішувалися в одному файлі. "Контроллер" відповідає за HTTP-рівень, Service за бізнес-правила й валідацію, Store за роботу з даними в пам'яті, а модель задачі за саму сутність. Такий поділ робить код простішим для читання, навіть якщо саме завдання невелике.

Якщо чесно, по коду, скоріше за все, буде видно, що хоч я і fullstack, основний акцент у цій роботі я робив саме на frontend-частині. Через це я вже бачу кілька своїх сліпих зон над якими варто попрацювати. 

## ✨ Що реалізовано

- Отримання списку задач з backend API
- Створення нової задачі
- Редагування назви задачі
- Видалення задачі
- Фільтрація списку задач
- Збереження вибраного фільтра в localStorage
- Дві візуальні теми для інтерфейсу
- Акуратна компонентна структура на frontend
- Розділення backend на окремі логічні шари

## 🧪 Що покрито тестами

- Отримання порожнього списку задач
- Створення нової задачі
- Оновлення існуючої задачі
- Видалення задачі
- Валідація порожнього title

Frontend теж підготовлений до нормального тестування, а основні критичні сценарії вже закладені так, щоб їх можна було легко розширити без болю і шаманства.

## 🚀 Локальний запуск

Для локального запуску потрібні два окремі термінали: один для backend, другий для frontend.

Backend:

```powershell
cd apps/backend
npm install
npm run dev
```

Frontend:

```powershell
cd apps/frontend
npm install
npm run dev
```

Після запуску очікувано:

- backend працює на `http://localhost:3001`
- frontend відкривається на `http://localhost:5173`
